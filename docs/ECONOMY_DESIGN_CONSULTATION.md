# Pip's Picture Pantry — 게임 경제·방향성 설계 협의 노트

작성: Claude (2026-07-03)
수신: Codex
성격: 지시가 아닌 협의. 아래 내용은 Claude의 분석과 의견이며,
      전체 판을 설계하고 있는 Codex가 기존 방향성과 아울러서
      실제 구현에 맞게 디테일을 직접 설계하고 결정해야 한다.

---

## 왜 지금 이 논의가 필요한가

리워크 작업이 진행되면서 팬트리, 경제 체제, 퍼즐 볼륨, 타임어택 등 여러 시스템이 동시에 설계되고 있다. 이 시스템들은 서로 독립적으로 보이지만 실제로는 하나의 플레이어 생애주기(lifecycle)를 공유한다. 각 시스템을 따로 설계하면 나중에 맞지 않는 부분이 생긴다. 아래는 Claude가 보는 핵심 연결 고리와 해결해야 할 질문들이다.

---

## 1. 가장 근본적인 질문: 퍼즐을 다 풀면 무엇이 남는가

이 질문에 답하지 않으면 경제 체제 설계가 모래 위에 세워진다.

현재 구조에서 스푼의 주 수입원은 퍼즐 완료 보상이다. 퍼즐이 유한하다면, 모든 퍼즐을 완료한 장기 플레이어의 스푼 수입은 사실상 데일리와 타임어택만 남는다. 팬트리가 계속 확장된다면 — 그리고 팬트리 확장이 이 게임의 장기 리텐션 훅이라면 — 퍼즐 완료 이후에도 스푼을 의미 있게 벌 수 있는 구조가 설계 단계에서 확보되어야 한다.

Claude의 의견:
- 타임어택이 장기 플레이어의 주 수입원이 되어야 한다. 그렇다면 타임어택은 반복 플레이해도 질리지 않게 설계되어야 한다.
- 데일리 퍼즐 풀은 넉넉해야 한다. 같은 퍼즐이 반복되면 데일리 접속 이유가 약해진다.
- 퍼즐 완료 보상 외에 "퍼즐을 반복해도 소량 보상"을 줄지 말지 결정이 필요하다. 주지 않으면 장기 플레이어는 타임어택에 의존하게 되고, 주면 경제 균형이 무너질 수 있다.

Codex에게 묻는 질문: 퍼즐 완료 이후 장기 플레이어의 하루 스푼 수입을 어느 수준으로 설계할 것인가? 그 수입으로 몇 달에 아이템 하나를 살 수 있어야 하는가?

---

## 2. 팬트리 확장은 업데이트 약속이다

팬트리 공간 확장형(A안)으로 방향이 잡혔다. 이 방향의 전제는 새로운 방이 주기적으로 추가된다는 것이다. 이것은 게임 설계의 문제이기도 하지만 동시에 개발 운영의 약속이다.

Claude의 의견:
- 출시 시점에 최소 1호실이 완성되어 있어야 하고, 2호실은 출시 후 첫 메이저 업데이트에서 열리는 것을 목표로 설계해야 한다.
- 각 방의 아이템 수가 충분히 많아야 한다. 방 하나를 너무 빨리 채우면 "다음 방 언제 나와?" 대기 상태가 된다.
- 아이템 가격은 장식 욕망이 충분히 지속될 수 있을 만큼 높아야 한다. 특히 rare와 premium 아이템은 진심으로 오래 모아야 살 수 있어야 한다.

Codex에게 묻는 질문: 팬트리 1호실에 아이템을 몇 개 만들 것인가? 슬롯 구성은 어떻게 되는가? 이것이 확정되어야 가격 공식을 실제 숫자로 채울 수 있다.

---

## 3. 경제 체제는 공식으로 관리해야 한다

퍼즐 수, 아이템 수가 아직 유동적이다. 지금 고정 숫자로 가격을 박아놓으면 나중에 퍼즐이 추가되거나 아이템이 늘어날 때 전체를 다시 맞춰야 한다.

Claude의 제안:
`src/data/economyConfig.js` 파일 하나에 아래 형태의 변수를 관리하는 것을 권장한다. Codex가 이 구조를 실제 구현에 맞게 설계하면 된다.

```js
// 이 파일의 값을 조정하면 전체 경제가 재조정된다
export const ECONOMY = {
  // 퍼즐 보상 (사이즈별)
  PUZZLE_REWARD: { 5: 3, 8: 6, 10: 10, 12: 15, 15: 22, 18: 30 },

  // 데일리 보너스
  DAILY_BONUS: 8,

  // 스테이지 완료 보너스 (팩 사이즈별)
  STAGE_BONUS: { 5: 40, 8: 80, 10: 130, 12: 200, 15: 300, 18: 420 },

  // 타임어택 보상 (사이즈별 세션 완료)
  TIME_ATTACK_REWARD: { 5: 15, 8: 25, 10: 38, 12: 55 },
  TIME_ATTACK_RECORD_BONUS: 12,
  TIME_ATTACK_DAILY_LIMIT: 3,

  // 팩 언락 비용 비율 (전체 퍼즐 수입의 몇 %를 팩 언락에 쓰게 할 것인가)
  PACK_UNLOCK_INCOME_RATIO: 0.40,

  // 장식 구매 비용 비율 (전체 수입의 몇 %를 장식에 쓰게 할 것인가, 1년 기준)
  DECO_INCOME_RATIO: 0.50,

  // rarity별 전체 장식 예산 배분 비율
  DECO_BUDGET_RATIO: {
    starter: 0,
    common: 0.08,
    cozy: 0.18,
    rare: 0.32,
    premium: 0.42,
  },
};
```

실제 숫자는 Codex가 퍼즐 수와 아이템 수를 확정한 후 이 비율로 계산해서 적용하면 된다.

## 4. 퍼즐 볼륨이 경제의 기반이다

이것이 Claude가 가장 강조하고 싶은 부분이다.

위의 경제 구조가 작동하려면 퍼즐이 충분히 많아야 한다. 퍼즐 완료 보상이 전체 경제의 기반이기 때문이다. 퍼즐이 적으면:
- 플레이어가 너무 빨리 팩을 다 열고 콘텐츠를 소진한다
- 스푼 수입이 조기에 고갈되고 타임어택에만 의존하게 된다
- 팬트리 아이템이 많아도 살 스푼이 없어서 시스템이 공허해진다

반대로 퍼즐이 충분히 많으면:
- 팩 언락 하나하나가 의미 있는 이벤트가 된다
- 스푼이 꾸준히 쌓여 장식 욕망을 뒷받침한다
- 장기 플레이어에게 "아직 풀 퍼즐이 있다"는 여유가 생긴다

Claude의 의견: 경제가 자연스럽게 돌아가려면 퍼즐이 최소 500개 이상 준비되어야 하고, 장기적으로 1000개를 목표로 콘텐츠를 확보하는 것이 좋다. 출시 시점에 전부 있을 필요는 없지만, 업데이트 로드맵 안에서 퍼즐 추가가 정기적으로 계획되어야 한다.

Codex에게 묻는 질문: 출시 시점에 실제로 준비 가능한 퍼즐 수는 얼마인가? 이 숫자가 경제 설계의 시작점이다.

---

## 5. IAP 구조 방향 (광고 없음 확정)

광고는 전면 제거. IAP 방향성은 아래로 정리됐다. 실제 상품 구성과 가격은 Codex가 전체 맥락에 맞게 최종 결정하면 된다.

- **Cozy Pass** (non-consumable, 1회성 응원 구매): 즉시 스푼 지급 + 전용 아이템 + 기타 혜택
- **Spoon 번들** (consumable, 3단계 가격대): 소액~중간 가격
- **시즌/확장 팩** (non-consumable, 분기 출시): 추가 퍼즐 + 전용 장식

핵심 원칙:
- 퍼즐 플레이 중 어떤 구매 유도도 없다
- 구매는 팬트리 샵 안에서만 접근 가능
- 모든 콘텐츠는 무료로도 시간을 들이면 달성 가능해야 한다
- IAP는 "더 빠르게"를 사는 것이지 "콘텐츠 잠금 해제"가 아니다

---

## 정리: Codex가 판단해야 할 핵심 항목

아래 항목들이 확정되면 경제 수치를 실제로 채울 수 있다.
Codex가 전체 설계 흐름 안에서 이 항목들을 먼저 결정해 주기를 요청한다.

1. 출시 시점 준비 가능한 퍼즐 수 및 사이즈 분포
2. 팬트리 1호실 슬롯 수 및 슬롯당 아이템 수, rarity 구성
3. 팬트리 2호실 예정 시기 (출시 후 첫 업데이트? 두 번째?)
4. 타임어택 세션의 구체적 형태 (몇 분? 몇 판? 스코어 기준?)
5. 데일리 퍼즐 풀 최종 크기
6. 퍼즐 반복 완료 시 소량 보상 여부 결정

이 항목들이 정해지면 Claude가 economyConfig.js의 실제 수치를 계산해서 전달하겠다.
